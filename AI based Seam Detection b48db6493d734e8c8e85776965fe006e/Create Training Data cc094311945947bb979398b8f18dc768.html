Step 4 : Splitting data to train and test

Training Data

Training data should contain the images the model should learn from.

By convention you should use 80 or 70% of your entire data for training. For this purpose, randomly copy 80% of total data and their annotation to the /train folder

Testing Data

Testing data is the images the model has never seen before.

Usually consists of 20% of the entire dataset. After the training is completed, the model is tested on this data to determine the accuracy and performance. Copy 20% of total data and their annotation to /test folder.

Step 5 : Create the data folder for training

This data directory will be volume mounted to the docker container while retraining the model. The container expects the data arranged specifically in the filesystem.

In the root FS, create a folder called data, this will contain all images and labels required for training the images.

data should have two subfolders - train and test containing the training and testing data respectively. It should also contain a file label_map.pbtxt a file consisting of all the labels of different classes for detection. For our use case, there is only one class called seam.

You can safely resue this file if you only want to train the model on more images.

train subfolder will contain the images and labels for each images in separate folders, this is similar for test data.
